# London Temperature Signal Engine

Минимальный прототип прогноза максимальной температуры Лондона для Polymarket. Скрипт полностью автономно:

1. Тянет прогнозы максимальной температуры (модели ECMWF/ICON) через Open-Meteo.
2. Агрегирует медиану/консенсус на завтра.
3. Сравнивает с оценкой Polymarket по указанному рынку (парсит название исходов, чтобы извлечь значения температуры).
4. Вместо фактического входа просто выводит прогноз и сигнал в консоль и сохраняет запись в `logs/predictions.ndjson`.

## Быстрый старт

```bash
cp .env.example .env
# Отредактируй .env, подставив ID рынка (например, из GraphQL или REST API Polymarket).
npm install
npm run predict
```

## Конфигурация

Файл `.env` может содержать:

- `POLYMARKET_MARKET_ID` — обязательный. ID рынка, на основе которого происходит аналитика.
- `POLYMARKET_BASE_URL` — необязательный. По умолчанию `https://api.polymarket.com/v0`.
- `VALUE_TEMPERATURE_DIFF_THRESHOLD` — сколько градусов разницы нужно, чтобы считать сигнал сильным (по умолчанию `1.2°C`).
- `OUTCOME_TEMPERATURE_MAP` — необязательный JSON-объект (в одну строку). Позволяет задать, какие значение температуры соответствует конкретному исходу (например, `{"Over 24°C": 24}`).

## Как работает

1. Скрипт запрашивает Open-Meteo по моделям (ECMWF, ICON). Для каждой рассчитывает максимальную температуру на следующий день.
2. Полученные значения агрегируются (медиана, среднее, std-dev, spread). Чем меньше spread — тем выше доверие к прогнозу.
3. Запрашивается рынок Polymarket по заданному `POLYMARKET_MARKET_ID`. Если API возвращает исходы, скрипт пытается извлечь из их названий значения температуры (парсинг чисел и «over/under»).
4. Вычисляется рыночное ожидание и сравнивается с нашим прогнозом. Если разница больше порога (`VALUE_TEMPERATURE_DIFF_THRESHOLD`), скрипт выдаёт сигнал `YES`, иначе — `NO`. Текст сигнала и summary выводится в консоль и пишется в `logs/predictions.ndjson`.

## Дальше

Когда протестируешь модель (например, наблюдая за реальной температурой и выводами), мы сможем:

- Добавить постоянный цикл (cron/worker), чтобы прогноз приходил каждый день.
- Подключить твой кошелёк и начать фактическое выставление ставок (stop-loss, трэйлинг и т.д.).
- Добавить фиксацию результатов, анализ точности и расчёт профита.

### Observed data source

Наблюдаемая температура теперь подтягивается только из разрешённых исходов Polymarket. Скрипт `scripts/fetch_observed.js` подставляет дату в slug `https://polymarket.com/event/highest-temperature-in-london-on-<month>-<day>-<year>` и парсит текст страницы (включая фразу «reached exactly XX°C»). Логи (`logs/observed-london.ndjson`) теперь отображают Polymarket как источник. Если нужно поменять slug-паттерн, редактируй `EVENT_SLUG_PREFIX` и `MONTH_FORMATTER` там же.
